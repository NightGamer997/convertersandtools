🔐 Hash-Algorithmen erklärt:

MD5 (Message Digest Algorithm 5)

  • Hash-Länge: 128 Bit (32 Hex-Zeichen)
  • Geschwindigkeit: Sehr schnell
  • Sicherheit: ⚠️ Nicht mehr sicher für kryptographische Zwecke
  • Anwendung: Checksums, nicht-sicherheitskritische Prüfsummen

SHA-1 (Secure Hash Algorithm 1)

  • Hash-Länge: 160 Bit (40 Hex-Zeichen)
  • Geschwindigkeit: Schnell
  • Sicherheit: ⚠️ Veraltet, seit 2017 nicht mehr empfohlen
  • Anwendung: Legacy-Systeme, Git (für Commit-IDs)

SHA-256 (Secure Hash Algorithm 256)

  • Hash-Länge: 256 Bit (64 Hex-Zeichen)
  • Geschwindigkeit: Moderat
  • Sicherheit: ✅ Sehr sicher, aktueller Standard
  • Anwendung: Passwort-Hashing, Blockchain, SSL/TLS-Zertifikate

SHA-512 (Secure Hash Algorithm 512)

  • Hash-Länge: 512 Bit (128 Hex-Zeichen)
  • Geschwindigkeit: Schnell auf 64-Bit-Systemen
  • Sicherheit: ✅ Sehr sicher, höchste Sicherheit
  • Anwendung: Hochsichere Anwendungen, digitale Signaturen

⚠️ Wichtige Hinweise:

Keine Entschlüsselung: Hash-Funktionen sind Einweg-Funktionen - ein Hash kann nicht zurück in den Original-Text umgewandelt werden.

Nicht für Passwörter: Für Passwort-Speicherung sollten spezielle Algorithmen wie bcrypt, Argon2 oder PBKDF2 verwendet werden, die Salting und Key-Stretching unterstützen.

Gleicher Input = Gleicher Output: Der gleiche Text erzeugt immer den gleichen Hash (deterministische Funktion).

Kollisionen: MD5 und SHA-1 sind anfällig für Kollisionsangriffe. Verwende SHA-256 oder höher für sicherheitskritische Anwendungen.
